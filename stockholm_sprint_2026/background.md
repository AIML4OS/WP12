# Background
## WP12 Stockholm Sprint on Large Language Models

The Stockholm sprint is organised as part of Work Package 12 (WP12) in the AIML4OS project. WP12 focuses on the use of generative AI and Large Language Models (LLMs) in official statistics, including practical prototypes, architectural considerations and guidance for implementation.

The sprint will take place at Statistics Sweden (SCB) in Stockholm, 16-18 June 2026. It builds on earlier WP12 work, including the Lisbon sprint in June 2025, where several LLM-based prototype examples were developed and later used as input to WP12 deliverable material.

## About AIML4OS

AIML4OS, One-Stop-Shop for Artificial Intelligence and Machine Learning for Official Statistics, aims to support the use of AI and machine learning across the European Statistical System. The project brings together work on infrastructure, methodology, standards, implementation frameworks and practical use cases so that national statistical institutes can explore, develop and reuse AI/ML solutions in a coordinated way.

The overall goal is to help move AI/ML from experimentation towards practical and responsible use in official statistics. This includes developing shared resources, prototypes, guidance and environments that can support quality, transparency, reproducibility, data protection and collaboration across organisations.

## Purpose of WP12

WP12 explores how LLM-based approaches can be used in the context of official statistics. The work includes both practical prototype development and reflection on how such systems can be implemented in a way that supports reuse, data protection, transparency, evaluation and operational relevance.

A recurring theme in WP12 is that useful LLM-based systems depend on the surrounding architecture, including tools, APIs, prompting strategies, retrieval mechanisms, data and metadata sources, orchestration frameworks, interfaces, hosting choices and evaluation routines.

## Purpose of the Stockholm sprint

The purpose of the Stockholm sprint is to make concrete progress on WP12 prototype work and to capture material for upcoming WP12 deliverables. The sprint should therefore be seen both as a collaborative development sprint and as a structured documentation effort.

The aim is to:

- develop or extend LLM-based prototype material
- explore selected use cases relevant for official statistics
- test practical implementation approaches in a shared environment
- document architectural choices, limitations and lessons learned
- capture material that can support upcoming WP12 deliverables
- strengthen the link between prototype work and evaluation of outputs

A key lesson from previous work is that prototype results should be documented while the work is carried out, not only afterwards. Sprint teams should therefore capture short notes on use case purpose, data, technical setup, evaluation approach, limitations and possible reuse.

## Expected outputs

The expected outputs from the sprint may include:

- prototype code or notebooks
- short README files for each use case
- documentation of selected technical approaches
- notes on evaluation and comparison with alternative approaches
- reflections on reuse, data access, architecture and limitations
- draft material for upcoming WP12 deliverable work

The sprint should also help clarify which use cases are promising for further development after Stockholm.

## Current use-case focus

The latest WP12 planning meeting identified two main use cases to explore during the sprint:

1. Web Corner / agentic web scraping
2. Metadata management using AI

Other use-case ideas have also been discussed, including summarising, translating and tagging statistical output reports, and new approaches for coding and classification. These may still be relevant depending on participant interest, available data and team capacity during the sprint.

More information about the planned use cases is available in `use-case-planning.md`.
